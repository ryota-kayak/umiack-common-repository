# Umiack Web Common Repository

このリポジトリは、Umiackのカヤックツアー等のWebサイト向け共通アセット、WordPress用HTML/CSS、および画像最適化ツールを管理するためのものです。

## プロジェクトの設計原則（Iron Rules）

開発にあたって、以下のプロフェッショナルな設計原則を厳守してください。

### 1. ソースアセットの不変性 (Source Invariance Policy)
*   **`src/` 内のオリジナルデータは絶対に直接加工（上書き）してはいけません。**
*   撮影時の写真（JPEG等）が横向きであったとしても、`src/` 内ではそのままの状態で保管します。
*   ファイル名の変更も、管理台帳（`images.json`等）の更新を伴わない限り、慎重に行うべきです。

### 2. ビルド時の自動変換 (Build-time Transformation)
*   Web表示に必要な全ての最適化（WebP変換、リサイズ、回転処理）は、ビルドプロセス（`dist/` への出力時）に行います。
*   画像の回転は、オリジナルファイルに含まれる「Orientation（回転タグ）」をビルドスクリプトが読み取り、物理的なピクセル回転として出力します。
*   出力された画像からは、不要な回転タグは自動的に除去されます。

## ディレクトリ構造

*   `src/`: ソースファイル。
    *   `src/wordpress/`: WordPress用ページテンプレートと画像ソース。
    *   `src/server/common/`: サイト共通のアセット（CSS, JS, 共通画像）。
*   `dist/`: 公開用ファイル。ビルドスクリプトによって自動生成されます。
*   `scripts/`: ビルド・最適化ツール。
    *   `optimize-images.mjs`: 高機能画像最適化・回転処理コア。
    *   `build.sh`: プロジェクト全体のビルドをオーケストレートするシェルスクリプト。

## 開発ガイド

### 画像の追加
1.  各ツアーフォルダの `img/` ディレクトリにオリジナルの写真を配置します。
2.  同フォルダの `images.json` にファイル名と `alt` テキストを追記します。
3.  ビルドを実行します（下記参照）。

### ビルドの実行
以下のコマンドで、全ての画像が最適化され、`dist/` フォルダに必要なファイルが生成されます。

```bash
bash ./scripts/build.sh
```

## 依存関係
このプロジェクトは画像処理に `sharp` を使用しています。環境には `Node.js` (v20以上推奨) が必要です。
