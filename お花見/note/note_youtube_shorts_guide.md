# AIでも解けなかった、WordPress（Cocoon）でYouTube Shortsを縦長表示させる攻略ガイド

「記事の中に流行りのYouTube Shorts（ショート動画）を埋め込もうとしたのに、どうやっても両サイドに巨大な黒帯が出て、従来の横長サイズになってしまう……」

あなたは今、こんな問題に直面していませんか？
実はこの問題、高性能なAIに聞いても、すぐには正しい答えが返ってきません。

なぜなら、そこにはWordPressの人気テーマならではの**「ある親切な機能」が「見えない罠」として暴走している**からです。

本記事では、この問題を根本的に解決する「コピペで使える完璧なコード」と、「なぜそんな現象が起きるのか？」という専門的なフロントエンドの仕組みまでを、初心者にも分かりやすく完全に網羅して解説します。

ご自身のブログや企業サイトのUIを決定的に改善し、貴重な過去記事のレイアウトを一切破壊せずに済む、まさに「現場のベストプラクティス」と呼べるテクニックです。

## 1. どんな問題が起きるのか？（YouTube埋め込みの致命的な2大トラップ）

YouTube ShortsのリンクをWordPressに貼り付けると、本来は「スマホサイズの縦長（アスペクト比 9:16）」で隙間なくピッタリと表示されてほしいはずです。

しかし、そのままプレビューを見てみると、**重大で致命的な「2つのトラップ」** が発動してしまいます。

### 🚨トラップ① 巨大な黒帯と、不格好な「極小表示」
まずは見た目の問題です。動画の左右に真っ黒な帯（レターボックス）が出現し、強引に「横長（アスペクト比 16:9）」の枠の中に、すきま風が吹くように極小の縦長動画が表示されるという、非常に不格好な状態になってしまいます。

👉 `▲ 【対策前】横長の巨大な黒帯が出てしまい、動画が小さく表示されてしまう失敗例`

### 🚨トラップ② 「全画面表示」にすると画質がボロクズになる（致命傷）
実は、デザインの崩れ以上にこれが**最もユーザーの離脱を招く致命的な問題**です。

スマホでこの「横長枠の中の、極小縦型動画」を見たユーザーは、小さくて見えにくいため、動画の右下にある「全画面拡大アイコン（斜めの矢印）」をタップして拡大しようとします。
すると、確かにスマホの画面いっぱいに縦長で動画が開くのですが……**動画全体が非常に粗く、ボケボケのモザイクのような画質**になってしまうのです。

これは、YouTubeのシステムが「こんな小さな枠で再生されるなら、通信量節約のために一番低い解像度（低画質）のデータを送ろう」と自動判定してしまうことが原因です。
低画質のまま読み込まれた動画を無理やり全画面に引き伸ばすため、悲惨な画質になります。綺麗な画質で魅力を伝えるためには、**「最初からWebページ上で適切な縦長の大きさとして認識させる」**必要が絶対にあるのです。

## 2. なぜAIに聞いても解決しないのか？

ここでAIに「YouTubeを縦長に直して」と質問すると、大抵は以下のようなCSSを提案してきます。

```css
/* AIがよく提案してくる（失敗する）解決策 */
.youtube-shorts iframe {
  width: 100%;
  aspect-ratio: 9 / 16;
}
```

結論から言うと、これをコピペしても現実は**ピクリとも動きません**。
それどころか、表示がさらに崩れてしまうこともあります。ここが「AIでも簡単に解けない罠」の入り口です。

## 3. 明確な原因：WordPress本体ではなく「Cocoon」の独自機能

なぜ、完璧と思えるCSSが全く効かないのでしょうか？
結論から言うと、この「強大な見えない壁」を作っている直接の原因は、WordPress本体の仕様ではなく、人気の無料テーマ**「Cocoon」の独自の親切機能**にあります。

WordPress本体の標準仕様では、自分が書いたHTMLの `<iframe>` タグに対して、勝手に別の邪魔なタグを強制的に付け足すような大胆な改変は行いません。

しかし、親切であるはずのCocoonは、記事がユーザーに表示される直前のタイミングで**「本文内にあるすべてのYouTube埋め込み（iframe）を片っ端から探し出し、強制的に以下のようなHTMLコードで強引に囲い込んで出力する」**という独自の裏処理（フィルターフック）を行っています。

```html
<!-- あなたが書いたコードはこれだけなのに... -->
<iframe src="https://www.youtube.com/..."></iframe>

<!-- 実際のウェブ上ではCocoonが勝手にこう変換している！ -->
<div class="video-container">
  <div class="video">
    <iframe src="https://www.youtube.com/..."></iframe>
  </div>
</div>
```

そして、この追加された `<div class="video-container">` には、Cocoon側から強制的なCSSがかけられています。

```css
/* Cocoonが裏でかけている強力なCSS */
.video-container .video {
  padding-bottom: 56.25%; /* これが16:9の正体！ */
  height: 0;
}
```

### padding-bottom: 56.25% という過去の遺物
これは「Padding-Bottom Hack」と呼ばれる、横長動画をスマホサイズに合わせて綺麗に縮めさせるための旧来のシステムです。「初心者の方がYouTubeをいかに雑に貼り付けても、スマホ画面からはみ出さないようにしてあげる」という、Cocoon開発者の素晴らしい親切設計です。

しかし、この「16:9強制ギブス」がすべての動画に自動装着されてしまうがゆえに、あなたが書いたCSSでいくら中身の動画を縦長（9:16）にしようと足掻いても、外側のギブスが横長である以上、絶対に「両脇に黒帯のついた不格好な横長動画」になってしまうのが、この問題の明確な全貌です。

## 4. 陥りがちな罠：安易に機能をオフにしてはいけない

「なるほど、じゃあそのCocoonテーマの設定でお節介機能をオフにすればいいんだ！」
実は、Cocoonの設定画面（「Cocoon設定」＞「動画」）にある「動画のレスポンシブ再生を有効にする」のチェックを外せば、この強制ギブスは簡単に外れます。

**しかし、絶対にオフにしてはいけません。**

これをシステム全体でオフにしてしまうと、あなたが過去に書いた記事にある「通常の横長YouTube動画」や、これから貼る普通の動画まで、すべてスマホ画面でレスポンシブにならず、右半分が画面外にはみ出して見切れる大惨事になります。

つまり私たちの目指すべき真のゴールは、
**「過去の横長動画は今まで通り綺麗に表示させつつ、指定した箇所のショート動画だけを完璧な縦長（高画質）にする」**という極めて局所的でスマートなハックなのです。

## 5. 完全解決策：コピペで終わるベストプラクティス

では、具体的な解決手順に入ります。
やることは「HTMLに1単語クラスを追加する」ことと、「CSSに専用のコードをコピペする」だけです。

### STEP 1：CSSに専用コードを追加する
WordPressの「外観」＞「カスタマイズ」＞「追加CSS」、またはお使いの子テーマの `style.css` に、以下のコードをそのままコピー＆ペーストしてください。

```css
/* =========================================
   YouTube Shorts（スマホ縦長動画 9:16）完全対応
   ========================================= */

/* ① ショート動画専用の「縦長の外枠」を作る */
.youtube-shorts {
  width: 100%;
  max-width: 360px; /* PC閲覧時に巨大化しすぎるのを防ぐ設定 */
  margin: 2em auto; /* 画面の中央にスッキリ配置 */
  aspect-ratio: 9 / 16; /* ここで「縦長」の確固たる箱を確保する！ */
}

/* ② Cocoonの「16:9強制ギブス」をこの枠内"だけ"解除する（超重要） */
.youtube-shorts .video-container,
.youtube-shorts .video-container .video {
  padding-bottom: 0 !important; /* 横長の比率拘束を破壊 */
  height: 100% !important; /* 箱の高さを縦長に完全に追従させる */
}

/* ③ YouTube動画本体の見栄えを整える */
.youtube-shorts iframe {
  display: block;
  width: 100%;
  height: 100%; /* 親の縦長の箱に隙間なくフィットさせる */
  border: 0;
  border-radius: 12px; /* スマホ風に角を少し丸めて洗練させる */
  box-shadow: 0 4px 15px rgba(0,0,0,0.1); /* ふんわりと影をつけて浮かせる */
}
```

### STEP 2：記事（HTML）に動画を貼る
記事投稿画面で、ショート動画を埋め込みたいときは「カスタムHTMLブロック」を使用し、以下のように囲って書きます。

```html
<div class="youtube-shorts">
  <iframe src="ここにショート動画の共有用埋め込みリンク" title="YouTube video player" allowfullscreen></iframe>
</div>
```

**【設定はたったこれだけです！】**

たったこれだけで、通常の横長動画には一切の影響を与えず、`<div class="youtube-shorts">` で囲った部分だけが、PCで見てもスマホで見ても非常にスマートな縦型動画として表示されるようになります。（初めから適正なサイズで読み込まれるため、全画面表示に切り替えても高画質のままです！）

## 6. 専門的解説：なぜこのコードで完璧に動くのか？

最後に、この解決策のロジック的な良さを解説します。

1. **スコープ（影響範囲）の限定**
   ただ単に `.video-container` を解除するのではなく、`.youtube-shorts .video-container` と指定することで、テーマ全体の親切なシステムを殺すことなく、この特定のコンテナ内でのみ `padding-bottom: 56.25%` の呪縛を解明しています。
2. **AspectRatioの正しい適用先（AIとの違い）**
   AIがミスを犯しがちなのは「iframe自体に直接アスペクト比をかけようとする」点です。
   iframeのような外部要素は高さ計算が不安定になりがちです。本手法では、まず確固たる外箱（`.youtube-shorts`の`div`）に `aspect-ratio: 9/16` を指定し、完璧な「縦の空間」を先に強制的に掘り出しています。
3. **高さの強制同期**
   そして、呪縛を解かれた中間の `video-container` と `iframe` に対して `height: 100%;` を指定し続けることで、一番外側に作られた「完璧な縦長の箱」の内側に、液体のようにピッタリとiframeを満たすことに成功しています。

これが、最新のCSS仕様（`aspect-ratio`）と旧来のハック（`padding-bottom`）が同居する現代のWordPressにおける、最も安全なフロントエンドのベストプラクティスです。
